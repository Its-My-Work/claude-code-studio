# План миграции с Claude Code на Kilo Code

## Краткий обзор текущей архитектуры

Проект "Claude Code Studio" представляет собой веб-интерфейс для Claude Code CLI, позволяющий:
- Чатиться с AI через веб-UI
- Управлять задачами в Kanban-стиле
- Запускать автономные задачи
- Интегрировать через Telegram Bot
- Поддерживать MCP-серверы
- Управлять сессиями и контекстом

Основная интеграция с Claude Code осуществляется через:
- `claude-cli.js` - класс для вызова Claude CLI с аргументами (--print, --model, --max-turns и т.д.)
- `claude-ssh.js` - для удаленного выполнения через SSH
- Режимы: auto/planning/task, модели: haiku/sonnet/opus, agent modes: single/multi

## Список точек интеграции

| Файл | Сущность | Тип интеграции |
|------|----------|----------------|
| `claude-cli.js` | `ClaudeCLI.send()` | Запуск subprocess Claude CLI с аргументами, парсинг stream-json вывода |
| `claude-ssh.js` | `ClaudeSSH.send()` | SSH-подключение и запуск Claude на удаленной машине |
| `server.js` | `runCliSingle()`, `runMultiAgent()` | Формирование промптов, вызов ClaudeCLI, обработка ответов |
| `mcp-set-ui-state.js` | `set_ui_state` tool | Обновление UI состояния (mode, model, agent) через MCP |
| `telegram-bot.js` | Множественные вызовы | Интеграция с Telegram для чатов и задач |
| `mcp-task-manager.js` | Task management tools | Создание и управление задачами через MCP |

## Слой абстракции (AgentBackend)

Предлагаю ввести интерфейс `AgentBackend` с методами:
- `send({ prompt, sessionId, model, maxTurns, ... })` - отправка сообщения
- `getStatus()` - получение статуса агента
- `setMode(mode)` - установка режима (planning/task/auto)
- `setModel(model)` - установка модели
- `manageSession(sessionId, action)` - управление сессией

Текущая логика Claude интегрируется в `ClaudeBackend` - наследник `AgentBackend`.

## Проектирование Kilo-реализации

Предполагая аналогичный CLI-интерфейс Kilo Code:

KiloBackend будет:
- Вызывать `kilocode --print --model <model> --mode <mode> ...`
- Парсить stream-json ответы аналогично Claude
- Поддерживать MCP через `--mcp-config`
- Маппить режимы: planning -> architect, task -> code, auto -> orchestrator

Адаптация формата:
- Если формат ответов отличается, добавить трансформеры
- Для MCP проверить совместимость

### ⚠️ Критические подводные камни

#### 1. **Несовместимость режимов (ВЫСОКИЙ РИСК)**
- Claude: `planning`, `task`, `auto`
- Kilo: `architect`, `code`, `orchestrator`
- **Проблема**: маппинг режимов не тривиален — `auto` в Claude это не то же самое, что `orchestrator` в Kilo
- **Решение**: Нужна детальная документация поведения каждого режима в обоих системах перед маппингом

#### 2. **Формат stream-json ответов (ВЫСОКИЙ РИСК)**
- `claude-cli.js` парсит специфичный для Claude формат (строки 186-191)
- Kilo может использовать другой формат или структуру JSON
- **Проблема**: Если формат отличается, потребуется переписать парсер потока
- **Решение**: Получить примеры реальных ответов Kilo CLI перед началом разработки

#### 3. **MCP совместимость (СРЕДНИЙ РИСК)**
- Текущий код использует `--mcp-config` с кешированием по хешу (строки 84-120)
- Kilo может требовать другой формат конфигурации или другие флаги
- **Решение**: Проверить документацию Kilo на предмет MCP поддержки и формата конфигурации

#### 4. **Сессии и контекст (СРЕДНИЙ РИСК)**
- Claude использует `--resume <sessionId>` для восстановления сессии
- Kilo может использовать другой механизм или не поддерживать сессии вообще
- **Проблема**: Потеря функциональности восстановления сессий критична для UX
- **Решение**: Уточнить, как Kilo управляет сессиями и контекстом

#### 5. **Multi-agent режим (ВЫСОКИЙ РИСК)**
- `runMultiAgent()` (строки 2740-2845) полностью зависит от Claude
- Kilo может не поддерживать многоагентную оркестрацию или требовать другого подхода
- **Проблема**: Если Kilo не поддерживает multi-agent, нужно либо эмулировать, либо отключить
- **Решение**: Определить, есть ли в Kilo встроенная поддержка multi-agent

#### 6. **SSH интеграция (СРЕДНИЙ РИСК)**
- `claude-ssh.js` специфичен для Claude CLI
- Kilo может требовать другого подхода для удаленного выполнения
- **Решение**: Проверить, поддерживает ли Kilo SSH или требует другого механизма

#### 7. **Telegram Bot интеграция (НИЗКИЙ РИСК)**
- `telegram-bot.js` вызывает `runCliSingle()` и `runMultiAgent()`
- Если эти функции останутся совместимыми, Telegram Bot будет работать
- **Решение**: Убедиться, что новый `AgentBackend` имеет совместимый интерфейс

#### 8. **Инструменты и разрешения (СРЕДНИЙ РИСК)**
- Claude имеет специфичный набор инструментов: `Bash`, `View`, `SearchReplace`, `Write` и т.д.
- Kilo может иметь другой набор инструментов или другие имена
- **Проблема**: Строки 2315-2317 жестко кодируют инструменты для режимов
- **Решение**: Создать маппинг инструментов между Claude и Kilo

#### 9. **Обработка ошибок и rate limiting (СРЕДНИЙ РИСК)**
- `runCliSingle()` имеет сложную логику обработки rate limit (строки 2402-2448)
- Kilo может иметь другие коды ошибок или другой механизм rate limiting
- **Решение**: Изучить, как Kilo сообщает об ошибках и rate limiting

#### 10. **Система промптов и skills (НИЗКИЙ РИСК)**
- Система skills (`skills/*.md`) и построение системного промпта (строки 2092-2133) не зависит от Claude
- Должна работать с любым backend
- **Решение**: Убедиться, что `AgentBackend` принимает `systemPrompt` как параметр

## 🔬 ГЛУБОКИЙ АНАЛИЗ С УЧЕТОМ ДОКУМЕНТАЦИИ KILO

### 📊 Сравнение архитектур

#### Claude Code Studio
- **Интерфейс**: Веб-UI (Express.js + WebSocket)
- **Режимы**: `planning`, `task`, `auto`
- **Модели**: `haiku`, `sonnet`, `opus`
- **Agent modes**: `single`, `multi`
- **Инструменты**: `Bash`, `View`, `GlobTool`, `GrepTool`, `ListDir`, `SearchReplace`, `Write`, `ReadNotebook`, `NotebookEditCell`
- **Сессии**: Через `--resume <sessionId>` (встроено в Claude CLI)
- **MCP**: Через `--mcp-config` с кешированием по хешу
- **Вывод**: stream-json с парсингом по строкам

#### Kilo Code
- **Интерфейс**: TUI (Terminal User Interface) + `kilo run` для автономного режима
- **Режимы**: `architect`, `ask`, `debug`, `orchestrator`, + custom agent modes
- **Модели**: Любые модели через провайдеров (anthropic, openai и т.д.)
- **Agent modes**: Встроенная оркестрация через `/agents` команду
- **Инструменты**: Не документированы явно, но упоминаются в permission config
- **Сессии**: Через `/sessions`, `/continue`, `/fork` команды (встроено в TUI)
- **MCP**: Через конфиг `~/.config/kilo/opencode.json` или `.opencode.json`
- **Вывод**: Не документирован формат, но есть `--print-logs` флаг

### ⚠️ КРИТИЧЕСКИЕ РАЗЛИЧИЯ

#### 1. **Интерфейс (ВЫСОКИЙ РИСК)**
- **Claude**: Веб-UI с WebSocket потоком
- **Kilo**: TUI (интерактивный) или `kilo run` (автономный)
- **Проблема**: Kilo не имеет встроенного API для программного вызова
- **Решение**: Нужно либо:
  - Использовать `kilo run` как subprocess (аналогично Claude CLI)
  - Или создать wrapper для TUI (сложнее)
  - **Рекомендация**: Использовать `kilo run` как subprocess

#### 2. **Режимы (ВЫСОКИЙ РИСК)**
- **Claude**: `planning` (только анализ), `task` (выполнение), `auto` (оркестрация)
- **Kilo**: `architect` (планирование), `ask` (вопросы), `debug` (отладка), `orchestrator` (оркестрация)
- **Маппинг**:
  - `planning` → `architect` ✓ (логично)
  - `task` → `ask` или `debug`? (неясно)
  - `auto` → `orchestrator` ✓ (логично)
- **Проблема**: `task` режим в Claude это выполнение с инструментами, в Kilo нет прямого аналога
- **Решение**: Нужно определить, какой режим Kilo использует для выполнения кода

#### 3. **Модели (НИЗКИЙ РИСК)**
- **Claude**: Жестко кодированы `haiku`, `sonnet`, `opus`
- **Kilo**: Динамические модели через провайдеров
- **Преимущество**: Kilo более гибкий
- **Решение**: Использовать конфиг для маппинга моделей

#### 4. **Сессии (СРЕДНИЙ РИСК)**
- **Claude**: `--resume <sessionId>` в CLI
- **Kilo**: `/sessions`, `/continue`, `/fork` в TUI
- **Проблема**: Если использовать `kilo run`, сессии могут не работать
- **Решение**: Проверить, поддерживает ли `kilo run` флаг для восстановления сессии

#### 5. **MCP (СРЕДНИЙ РИСК)**
- **Claude**: `--mcp-config <path>` с JSON конфигом
- **Kilo**: `mcp` в `~/.config/kilo/opencode.json` или `.opencode.json`
- **Различие**: Kilo использует конфиг файл, а не флаг CLI
- **Решение**: Нужно создать конфиг файл вместо передачи флага

#### 6. **Инструменты (СРЕДНИЙ РИСК)**
- **Claude**: Явно перечислены в коде
- **Kilo**: Не документированы, но есть permission config
- **Проблема**: Неизвестно, какие инструменты доступны в Kilo
- **Решение**: Нужно получить список инструментов Kilo

#### 7. **Вывод (ВЫСОКИЙ РИСК)**
- **Claude**: stream-json с явной структурой
- **Kilo**: Не документирован формат вывода
- **Проблема**: Парсер потока может не работать
- **Решение**: Нужно получить примеры вывода Kilo

#### 8. **Автономный режим (СРЕДНИЙ РИСК)**
- **Claude**: Встроен в CLI
- **Kilo**: `kilo run --auto` с автоматическим ответом на вопросы
- **Различие**: Kilo требует явного флага `--auto`
- **Решение**: Использовать `kilo run --auto` для автономного режима

### 🎯 НОВЫЕ ПОДВОДНЫЕ КАМНИ

#### 1. **Отсутствие встроенного API (КРИТИЧЕСКИЙ)**
- Kilo не имеет встроенного API для программного вызова
- Нужно использовать `kilo run` как subprocess
- Это означает, что нельзя использовать TUI интерактивность
- **Решение**: Создать wrapper для `kilo run` аналогично `ClaudeCLI`

#### 2. **Конфигурация через файлы (СРЕДНИЙ РИСК)**
- Kilo использует конфиг файлы, а не флаги CLI
- Нужно создавать/обновлять конфиг файлы перед вызовом
- **Решение**: Создать `KiloConfigManager` для управления конфигом

#### 3. **Провайдеры и API ключи (СРЕДНИЙ РИСК)**
- Kilo требует явной конфигурации провайдеров
- Нужно передать API ключи через конфиг или переменные окружения
- **Решение**: Использовать `{env:VARIABLE_NAME}` синтаксис в конфиге

#### 4. **Отсутствие явного формата вывода (ВЫСОКИЙ РИСК)**
- Документация не описывает формат вывода `kilo run`
- Может быть текстовый вывод, JSON, или что-то еще
- **Решение**: Нужно экспериментально определить формат

#### 5. **Отсутствие явного списка инструментов (СРЕДНИЙ РИСК)**
- Документация не перечисляет доступные инструменты
- Только упоминаются в permission config
- **Решение**: Нужно получить список инструментов из документации или кода Kilo

#### 6. **Отсутствие явной поддержки multi-agent (СРЕДНИЙ РИСК)**
- Документация упоминает `orchestrator` режим
- Но не ясно, как он работает и как передать план
- **Решение**: Нужно экспериментально определить, как работает multi-agent

#### 7. **Отсутствие явной поддержки SSH (НИЗКИЙ РИСК)**
- Документация не упоминает SSH
- Возможно, нужно использовать другой механизм
- **Решение**: Проверить, есть ли в Kilo поддержка удаленного выполнения
