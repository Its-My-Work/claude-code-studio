# План: Реализация передачи режима через --mode вместо --auto

## Цель
Изменить реализацию KiloBackend и KiloRunner так, чтобы вместо флага `--auto` передавался флаг `--mode` с соответствующим значением режима (architect/code/debug/ask/orchestrator), в зависимости от выбранного пользователем режима

## Текущая реализация
- `KiloBackend.mapMode()` маппит режимы Claude на режимы Kilo:
  - planning → architect
  - task → code
  - auto → orchestrator
- `KiloRunner.run()` принимает `auto: true` и добавляет флаг `--auto`
- Режим Kilo не передается явно в CLI

## План изменений

### 1. Изменение KiloRunner.run()
- Убрать параметр `auto` из опций
- Добавить параметр `mode` (режим Kilo: architect/code/orchestrator)
- Вместо `--auto` передавать `--mode <mode>`

### 2. Изменение KiloBackend.send()
- Получить маппированный режим через `mapMode()`
- Передать этот режим в `runner.run()` вместо `auto: true`
- Убрать `auto: true` из вызова runner

### 3. Обновление интерфейса
- Убедиться, что режимы правильно маппятся во всех местах использования
- Проверить совместимость с существующими сессиями

## Файлы для изменения
- `backends/kilo-backend/kilo-runner.js` - изменить логику построения аргументов
- `backends/kilo-backend.js` - изменить вызов runner.run()

## Тестирование
- Убедиться, что CLI команды формируются правильно
- Проверить, что функциональность не изменилась для пользователя

## Риски
