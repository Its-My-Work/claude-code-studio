# План: Добавить выпадающий список моделей для KiloCode

## Цель
Когда AGENT_ENGINE=kilo (cliType === 'kilocode'), заменить текущее поле ввода модели на кнопку выпадающего списка в том же стиле, что и кнопка выпадающего списка проектов, обеспечивая более последовательный пользовательский интерфейс.

## Анализ текущего состояния
- **Десктоп**: Выбор модели использует `<input type="text" id="model-input" list="model-datalist">` в панели инструментов
- **Мобильный**: Выбор модели использует `<input type="text" id="mob-model-input" list="mob-model-datalist">` в мобильном листе
- **Выпадающий список проектов** служит эталонной реализацией с:
  - Кнопка: `proj-dd-btn` с иконкой (📁), именем и стрелкой (▼)
  - Меню: `proj-dd-menu` с полем поиска и списком элементов
  - Функции: `toggleProjDropdown()`, `renderProjDropdownList()`, `selectProjFromDropdown()`
- Оба десктопных и мобильных поля ввода модели заполняются через функцию `loadModels()`

## Предлагаемые изменения

### 1. Добавить HTML-структуру выпадающего списка моделей
Добавить новые элементы выпадающего списка в панель инструментов (десктоп) и мобильный лист, стилизованные идентично выпадающему списку проектов:
```html
<div class="model-dd-wrap" id="modelDdWrap">
  <button class="model-dd-btn" id="modelDdBtn" onclick="toggleModelDropdown()">
    <span class="model-dd-icon">🤖</span>
    <span class="model-dd-name" id="modelDdName">sonnet</span>
    <span class="model-dd-arrow">▼</span>
  </button>
  <div class="model-dd-menu" id="modelDdMenu">
    <div class="model-dd-search">
      <input type="text" id="modelDdSearch" placeholder="Поиск модели..." oninput="filterModelDropdown(this.value)">
    </div>
    <div class="model-dd-list" id="modelDdList"></div>
  </div>
</div>
```

### 2. Добавить CSS-стили
Дублировать стили выпадающего списка проектов с префиксом `model-dd-` вместо `proj-dd-`:
- `.model-dd-wrap`
- `.model-dd-btn` (с состояниями hover/focus)
- `.model-dd-menu` (позиционирование, анимация)
- `.model-dd-search`
- `.model-dd-list`
- `.model-dd-item` (с активным состоянием)
- и т.д.

### 3. Добавить JavaScript-функции
Реализовать логику выпадающего списка, зеркально отражающую выпадающий список проектов:

**toggleModelDropdown()**
- Переключать флаг `_modelDdOpen`
- Добавлять/удалять класс 'open' к кнопке и меню
- Фокусировать поле поиска при открытии
- Вызывать `renderModelDropdownList()`

**renderModelDropdownList(filter)**
- Фильтровать массив `availableModels` на основе ввода поиска
- Рендерить элементы списка с именами моделей
- Показывать активную модель с галочкой или выделением
- Обрабатывать пустое состояние

**selectModelFromDropdown(modelId)**
- Установить `curModel = modelId`
- Обновить отображаемое имя кнопки
- Закрыть выпадающий список
- Вызывать существующую логику `setOpt()` или обновления модели

**filterModelDropdown(value)**
- Вызывать `renderModelDropdownList(value)`

**closeModelDropdown()**
- Утилита для закрытия выпадающего списка (для клика снаружи и т.д.)

### 4. Обновить логику видимости
Изменить функцию `updateAgentModeVisibility()` для:
- Когда `cliType === 'kilocode'`:
  - Скрыть существующий `tb-group` модели (десктоп) и мобильную секцию модели
  - Показать `modelDdWrap` (десктоп) и мобильный выпадающий список модели
- Когда `cliType !== 'kilocode'`:
  - Показать существующие поля ввода модели
  - Скрыть выпадающие списки моделей

### 5. Обновить функцию loadModels()
Убедиться, что `renderModelDropdownList()` вызывается после загрузки моделей для первоначального заполнения выпадающего списка.

### 6. Обработать мобильную версию
Добавить соответствующую структуру мобильного выпадающего списка и убедиться, что `updateAgentModeVisibility()` обрабатывает оба метода выбора модели для десктопа и мобильного.

### 7. Синхронизировать отображение текущей модели
При загрузке страницы и изменениях модели обновлять `modelDdName` для показа текущего значения `curModel`.

- **Иконка**: Использовать 🤖 или подобный эмодзи, связанный с ИИ, для иконки выпадающего списка моделей
