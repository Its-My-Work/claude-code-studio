# План анализа системы логирования в Claude Code Studio

## Цель
Проанализировать текущую систему логирования и отображения логов в консоль сервера Claude Code Studio. Определить какие данные выводятся, выявить лишние/повторяющиеся логи и предложить улучшения для лучшей наблюдаемости и отладки.

## Текущие предположения
- Основные логи исходят из `server.js` (Express сервер) и `claude-cli.js` (обработка subprocess Claude CLI)
- Логи включают WebSocket сообщения, ошибки сервера, статусы subprocess
- Возможные проблемы: избыточное логирование чата, отсутствие уровней логов, неструктурированные сообщения

## Шаги анализа

### 1. Исследование кодовой базы логирования
- Прочитать `server.js` полностью для поиска всех console.log/error/warn
- Прочитать `claude-cli.js` для анализа парсинга JSON stream от subprocess
- Использовать grep для поиска всех мест логирования в проекте (console\., process\.stdout\.write, etc.)
- Проверить `public/index.html` на клиентские логи (хотя фокус на серверной консоли)

### 2. Категоризация текущих логов
- **Системные логи**: Запуск сервера, подключения WebSocket, ошибки сервера
- **Логи чата**: Входящие/исходящие сообщения, статус Claude subprocess
- **Логи ошибок**: Исключения, сбои subprocess, сетевые ошибки
- **Другие**: Аутентификация, база данных, MCP серверы

### 3. Анализ проблем
- Выявить повторяющиеся логи (например, частые статусы соединения)
- Найти лишние детали (слишком verbose вывод из Claude CLI)
- Оценить полезность каждого типа логов для отладки

### 4. Предложения по улучшению
- Внедрить структурированное логирование (winston или bunyan)
- Добавить уровни логов (debug, info, warn, error)
- Фильтрация логов по категориям
- Логирование в файлы для продакшена
- Улучшить форматирование для читаемости

## Риски и зависимости
- Claude CLI может выводить много JSON - нужно проверить, что именно парсится и логируется
- WebSocket трафик может генерировать много логов в режиме чата

## Критерии успеха
- Полный инвентарь всех типов логов в системе
- Выявлены конкретные проблемы (лишние/повторяющиеся логи)
- Предложены actionable улучшения с примерами кода

## Следующие шаги после плана
Запустить анализ путем чтения файлов и поиска паттернов логирования.
