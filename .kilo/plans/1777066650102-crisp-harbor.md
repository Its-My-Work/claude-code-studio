# План: Замена режимов на агенты Kilo в UI и бэкенде (с условной логикой)

## Цель
Обновить UI Claude Code Studio для отображения агентов Kilo Code (code, ask, plan, debug) при использовании KiloCode CLI, или оставить текущие режимы (auto, planning, task) при использовании Claude CLI без изменений. Сохранить тот же визуальный стиль и функциональность.

## Важное условие
**Режимы при использовании Claude CLI оставляем без изменений:** значения 'auto', 'planning', 'task' и их логика остаются как есть. Меняются только режимы при использовании KiloCode CLI — они заменяются на агенты Kilo: 'code', 'ask', 'plan', 'debug', 'orchestrator'.

Все изменения применяются только при обнаружении KiloCode CLI. Для Claude CLI поведение остается идентичным существующему.

## Анализ текущего состояния
- **UI**: Тулбар показывает группу "Режим" с кнопками: Auto, Plan, Task
- **Бэкенд**: Поле `mode` в sessions/messages использует значения: 'auto', 'planning', 'task'
- **Логика**: Режим влияет на system prompt в server.js (добавляет "MODE: PLANNING ONLY..." и т.д.)
- **CLI**: claude-cli.js запускает subprocess CLI без указания агента
- **Условная логика**: Нужно определить версию CLI и показывать соответствующие режимы

## Предлагаемые изменения

### 0. Добавление обнаружения версии CLI и получения списка агентов
- В claude-cli.js добавить функцию для определения версии CLI (kilo --version или claude --version)
- Определить, является ли CLI KiloCode (новый) или Claude (старый)
- Для KiloCode: добавить извлечение списка агентов аналогично моделям (возможно, hardcoded или через API)
- Сохранить результат обнаружения и список агентов в глобальной переменной или конфиге
- Обновить UI для динамического отображения режимов на основе версии CLI

### 1. Условное обновление кнопок режимов в UI (public/index.html)
- Если KiloCode: показать агенты ['code', 'ask', 'plan', 'debug', 'orchestrator'] с метками Code, Ask, Plan, Debug, Orchestrator — **только для KiloCode**
- Если Claude: оставить ['auto', 'planning', 'task'] с метками Auto, Plan, Task **без изменений** (сохраняем существующую логику полностью)
- Список агентов для KiloCode: hardcoded на основе документации (code, ask, plan, debug, orchestrator - deprecated)
- Сохранить метку "Режим" и тот же стиль для обоих случаев
- Аналогично обновить мобильный UI

### 2. Условное обновление обработки режимов в бэкенде (server.js)
- Добавить логику для двух наборов режимов:
  - Для Claude: ['auto','planning','task'] с существующими промптами — **без изменений**, оставляем как есть
  - Для KiloCode: ['code','ask','plan','debug','orchestrator'] с новыми промптами/агентами — **только для KiloCode**
- Обновить логику режим-промпт для KiloCode:
  - 'code': без специального промпта (поведение по умолчанию)
  - 'ask': добавить промпт только для чтения, ограничивающий инструменты
  - 'plan': добавить промпт, ориентированный на планирование
  - 'debug': добавить промпт, ориентированный на отладку
  - 'orchestrator': добавить промпт для оркестрации задач (deprecated)
- Режим по умолчанию: 'auto' для Claude (без изменений), 'code' для KiloCode

### 3. Условное обновление спецификации агента в CLI (claude-cli.js)
- Для KiloCode: добавить флаг --agent к аргументам subprocess на основе режима
- Сопоставить режимы с агентами: code→code, ask→ask, plan→plan, debug→debug, orchestrator→orchestrator — **только для KiloCode**
- Для Claude: оставить текущую логику без --agent флага **без изменений**
- Обеспечить совместимость с системой агентов Kilo CLI

### 4. Обновление схемы базы данных
- Изменения схемы не нужны (поле mode остается TEXT)
- Существующие сессии с режимами Claude ('auto','planning','task') будут продолжать работать без изменений
- Новые сессии KiloCode будут использовать агенты ('code','ask','plan','debug','orchestrator')

### 5. Обновление переводов
- Обновить строки i18n для новых меток режимов в public/index.html

## Шаги реализации
1. Добавить обнаружение версии CLI в claude-cli.js (определение KiloCode vs Claude)
2. Изменить public/index.html для условного отображения кнопок режимов (агенты KiloCode ИЛИ режимы Claude без изменений)
3. Обновить server.js: сохранить существующую логику Claude без изменений, добавить новые агенты только для KiloCode
4. Добавить условную поддержку флага --agent в методе send() claude-cli.js (только для KiloCode)
5. Протестировать рендеринг UI: Claude режимы остаются как есть, KiloCode показывает агенты
6. Проверить, что Claude CLI работает без изменений, а KiloCode получает правильные флаги --agent

## Риски
- KiloCode CLI может не поддерживать флаг --agent (требуется проверка)
- Различия в поведении агентов Kilo могут потребовать корректировки промптов
- Обратная совместимость: существующие сессии Claude должны работать без изменений
- Обнаружение версии CLI может быть ненадежным
- При ошибках определения версии fallback должен оставлять режимы Claude без изменений

## Верификация
- UI с Claude CLI: кнопки режимов Auto/Plan/Task отображаются и работают **без изменений**
- UI с KiloCode: кнопки агентов Code/Ask/Plan/Debug/Orchestrator отображаются корректно
- Клик по режимам обновляет curMode и отправляет в бэкенде для обоих типов CLI
- Бэкенд с Claude: использует существующие промпты 'auto'/'planning'/'task' без изменений
- Бэкенд с KiloCode: применяет новые агенты и правильные промпты
- Claude CLI: получает старые флаги, работает как раньше (без изменений)
- KiloCode: получает правильные флаги --agent для каждого агента
- Существующие сессии Claude продолжают работать без изменений
