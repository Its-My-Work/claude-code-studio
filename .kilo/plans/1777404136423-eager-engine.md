# План по удалению навыков (skills) из Claude Code Studio

## Цель
Полностью удалить функциональность навыков (skills) из проекта, поскольку Kilo имеет встроенные навыки и сам определяет их использование. Это упростит код и уберёт лишнюю функциональность.

## Предварительный анализ (что нужно удалить)
На основе поиска по коду, навыки используются в следующих местах:
- **UI элементы**: Панель навыков в правой боковой панели, чипсы в sessBarSkills, кнопки добавления/удаления навыков.
- **База данных**: Поле `active_skills` в таблице `sessions`.
- **Серверная логика**: Загрузка .md файлов из папки `skills/`, конкатенация в системный промпт.
- **Локализации**: Ключи для текстов навыков (skill.*, auto.title.*, etc.).
- **Файлы**: Папка `skills/` с .md файлами.
- **Конфигурация**: Локальные и глобальные настройки навыков.

## План выполнения
### Шаг 1: Анализ зависимостей
- Использовать grep для поиска всех упоминаний "skill" в коде (public/index.html, server.js).
- Проверить, что удаление не сломает другие функции (например, системный промпт должен работать без навыков).

### Шаг 2: Удаление UI элементов
- Удалить HTML для панели навыков (секция "Skills" в правой панели).
- Удалить чипсы навыков из sessBarSkills.
- Удалить кнопки и формы для добавления/редактирования навыков.
- Удалить обработчики событий для навыков (onclick, toggle functions).

### Шаг 3: Удаление серверной логики
- Удалить код загрузки навыков в server.js (чтение .md файлов, конкатенация в промпт).
- Удалить поля `active_skills` из базы данных (ALTER TABLE DROP COLUMN, миграция).
- Удалить функции для авто-определения навыков (detectSkillsFromText и т.п.).

### Шаг 4: Удаление локализаций
- Удалить ключи локализации, связанные с навыками (skill.*, auto.title.*, etc.) из TRANSLATIONS (en и ru).

### Шаг 5: Удаление файлов и папок
- Удалить папку `skills/` и все .md файлы.
- Удалить ссылки на skills в README или документации.

### Шаг 6: Тестирование
- Запустить сервер, проверить, что интерфейс работает без навыков.
- Отправить тестовое сообщение, убедиться, что промпт формируется корректно без навыков.
- Проверить базу данных на отсутствие ошибок.

## Риски и меры предосторожности
- **Риск**: Удаление может сломать системный промпт или другие функции. Решение: Перед удалением создать бэкап кода.
- **Риск**: Локальные навыки могут быть нужны для совместимости. Решение: Убедиться, что Kilo не использует их.
- **Аккуратность**: Удалять постепенно, тестируя после каждого шага.
- **Откат**: Если что-то сломается, можно восстановить из git.
