# План: Исправить обработку JSON events в kilo-runner.js

## Проблема

В текущей реализации `kilo-runner.js` не все события обрабатываются корректно:

1. **`step_start` не обрабатывается** — попадает в дефолтный `logger.debug` (строка 188)
2. **`step_finish` обрабатывается, но без данных о stepIndex**
3. **tool_use не логируется** — структура события отличается от ожидаемой
4. **text не логируется полностью**

## JSON event structure от Kilo

```
step_start:  {"type":"step_start","part":{"id":"...","type":"step-start"}}
step_finish: {"type":"step_finish","part":{"id":"...","type":"step-finish","reason":"tool-calls","tokens":{...}}}
tool_use:    {"type":"tool_use","part":{"type":"tool","tool":"bash","state":{...},"metadata":{...}}}
text:        {"type":"text","part":{"type":"text","text":"..."}}}
```

## План исправлений

### 1. Добавить обработку step_start
Добавить в `handleJsonEvent()`:
- Проверка `event.type === 'step_start'`
- Логирование с данными из `event.part`
- Извлечение stepIndex из `event.part.id` или `event.part.messageID`

### 2. Улучшить обработку step_finish
- Извлекать `reason` (tool-calls, stop, и т.д.)
- Извлекать токены: `event.part.tokens`
- Логировать cost

### 3. Улучшить обработку tool_use
- Проверять `event.part.tool` (не `event.part.name`)
- Извлекать: tool name, status, input, output, exit code
- Логировать время выполнения

### 4. Улучшить обработку text
- Проверять `event.part.text`
- Truncate для длинных текстов

### 5. Добавить обработку thinking (опционально)
- Тип события: `thinking` с `event.part.thinking`

## Файлы для изменения

- `backends/kilo-backend/kilo-runner.js` — функция `handleJsonEvent()`
